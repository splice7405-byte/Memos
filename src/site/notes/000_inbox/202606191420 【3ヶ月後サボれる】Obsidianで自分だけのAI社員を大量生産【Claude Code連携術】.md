- 00:00 <https://youtu.be/BT0sS_7y-bA?t=0m0s> AIの「回答が浅い」原因と解決策
    - AIが回答を浅く感じるのは、プロンプトのせいではなく、AIがあなたの「文脈（仕事内容や価値観）」を知らないため。
    - 「文脈」をAIに渡すことで、当たり障りのない一般論から自分専用の回答へと劇的に変化する。

- 01:38 <https://youtu.be/BT0sS_7y-bA?t=1m38s> 文脈の置き場所としてのObsidian
    - ObsidianはPC内に保存されるローカルのMarkdown形式ノートアプリ。
    - AIが読み込みやすいテキストで情報を蓄積でき、ノート同士をリンクで繋ぐことで「情報間の関係性」を可視化（グラフビュー）できる。

- 04:46 <https://youtu.be/BT0sS_7y-bA?t=4m46s> Claude Codeとの連携による「AI社員化」
    - Claude CodeはPC内のフォルダを読みながら作業できるAIエージェント。
    - Obsidianのノートが入っているフォルダ（ボルト）を丸ごとClaude Codeに参照させることで、あなたの前提知識を持った「自分専用のAI社員」として機能する。

- 07:57 <https://youtu.be/BT0sS_7y-bA?t=7m57s> Obsidianに蓄積すべき情報
    - 以下の5つを意識して少しずつノートを増やすとAIの精度が向上する。
        1. 自分のプロフィール
        2. 仕事の内容
        3. 過去の成果物
        4. 判断基準（大切にしている価値観）
        5. 今後の目標

- 08:27 <https://youtu.be/BT0sS_7y-bA?t=8m27s> 実演：文脈の有無による回答の変化
    - 文脈なし：教科書的で当たり障りのない回答しか得られない。
    - 文脈あり：過去の成果物や判断基準を反映した、その人専用の実践的な構成案が提案される。

- 11:02 <https://youtu.be/BT0sS_7y-bA?t=11m2s> さらに精度を上げる「ルールブック（cloud.md）」
    - 多くのノートがある場合、AIに「どのノートをどう読みに行くか」を指示するルールブック（`cloud.md`）を作成する。
    - 「文脈はノートに、読み方はルールブックに」という役割分担をすることで、AIの参照精度がさらに向上する。
